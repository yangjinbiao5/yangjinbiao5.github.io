String 的结构

String 的本质就是一个 char 数组

String 为什么用 final 修饰

  • final 修饰类，不可继承
    final 类中的所有成员方法都会被隐式地指定为 final 方法

  • final 修饰属性，属性是不可修改的
    对于一个 final 变量，如果是基本数据类型的变量，则其数值一旦在初始化之后便不能更改；如果是引用类型的变量，则在对其初始化之后便不能再让其指向另一个对象。

  • final 修饰方法，不可重写

String 是不可变类型

  • 用 final 修饰
  • value 数组权限控制，以及每次操作生成新对象
    每次对 String 都不是在原有的 value 数组上进行改动，而是生成一个新的数组

String 为什么被设计成不可变类型

1. 无法被继承 私有方法不能被重写（为了语义统一）
2. 每次操作生成新对象
3. 线程安全

使用 HashMap HashSet 等结构时，一定要注意使用不可变类型作为 key
假如要使用可变类型作为 key，要重写 hashCode 方法和 equals 方法

String 常量池

String 常量池位于方法区，一部分是编译期，一部分是运行时。

String 中的 intern 方法

public native String intern();

intern 方法用于扩充常量池
“abc”.intern()

首先检查常量池中是否有 “abc”，如果存在直接返回实例引用

如果不存在，会在常量池中新增 “abc”，然后返回实例引用
