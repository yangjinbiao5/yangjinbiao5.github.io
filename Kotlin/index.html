Kotlin

发表于 2024-07-31 更新于 2024-08-15 本文字数： 715 阅读时长 ≈ 3 分钟

特点

  1. 分号是可选的，kotlin 倾向于省略分号.
  2. REPL 读取 - 求值 - 输出 - 循环 命令行
  3. 可作为脚本运行
  4. 可编译为其他目标（Android,JavaScript,WebAssembly 等)
  5. 变量类型推断
  6. 语句和表达式可以脱离类和方法存在，当代码被编译或作为脚本执行时，kotlin 根据需要创建包装类和方法。
  7. try-catch 是可选的
  8. val 不可变，相当于 java 的 final，var 可变。如果可能，尽量选择 val 而不是 var
  9. == 相当于 java 的 equals () 方法，=== 相当于 java 的 == 。改进的点是 kotlin 会处理 null，不会出现空指针异常。( null == null 返回 true , null == “hi” 返回 false)
  10. 方便好用的字符串模板
  11. 原始字符串和多行字符串使用三个双引号包围，原始字符串可以不使用转义字符串，并且可以用作字符串模板。 （关于 trimMargin () 方法）
  12. 更多的表达式，更少的语句
  13. KISS 函数（Keep it simple ,stupid)
  14. 函数返回类型可推断
  15. Kotlin 使用 Unit 类，表示不包含任何信息的单例，对应于 Java 的 void 类型。所以 Kotlin 的函数必然有返回值，至少是 Unit。
  16. 默认参数
  17. 使用命名参数
  18. 可变数量的实参 (vararg 只有一个参数可以注释为 vararg, 并且要放到末尾), spread 运行符 (* 将数组的值作为 vararg 参数的离散值来展开)
  19. 解构
  20. 范围与迭代
  21. 遍历数组和列表
  22. when
